## Tarefa: Consolidar Edge Functions de mensageria

### Descrição
Identificadas 3 edge functions com o mesmo objetivo (enviar mensagens via security_fix_v2 + chat_only: false):
- `send-message` v3.1.0 — resolução interna de token, CLF1/JWT auth
- `speed-chat` v3.1.0 — igual ao send-message, dedicada ao Speed
- `lovable-proxy` v4.0.0 — proxy direto, aceita lovable_token no body

### Ação
- **Deletados**: `send-message` e `speed-chat` (código + deploy)
- **Mantido**: `lovable-proxy` como única edge function de chat
- **Atualizado**: `ProjectEditor.tsx` para usar `lovable-proxy` em vez de `send-message`

### Confirmação
Todas as 3 usavam `intent: "security_fix_v2"` + `chat_only: false` — idênticas em propósito.
