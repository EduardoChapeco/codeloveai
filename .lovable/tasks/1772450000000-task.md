## Tarefa: Cirius AI Editor — Refatoração Completa do Orquestrador e UI

### Descrição
No Cirius o orquestrador está sendo executado... esse orquestrador como funciona? Eu gostaria que fosse usado o Brainchain com fallback automático para OpenRouter (Claude)... melhore o UI visual para o design system... cards grandes.. criar editor visual para acompanhar...

O criador Cirius deverá fazer tudo automático... o cliente envia o primeiro prompt... o sistema faz etapa 1 de criar PRD... depois decide cores/fonte... estilo de design, o que será criado etc... mostra para aprovação (fala um resumo de páginas/tabelas que serão criados)... depois quero que crie elementos de progresso igual do Lovable... progresso, tasks, tarefas pendentes/a fazer com animação de relógio igual... realista.

### Subtarefas

1. **Orquestrador Brainchain + Fallback OpenRouter**
   - Usar Brainchain como motor principal de execução de tarefas
   - Fallback automático para OpenRouter (Claude) quando Brainchain indisponível
   - Manter lógica de fila e concorrência existente

2. **Fluxo Automático Completo**
   - Cliente envia prompt inicial → Sistema cria PRD automaticamente
   - Após PRD: decisão automática de cores, fontes, estilo de design
   - Resumo de aprovação: lista páginas, tabelas, componentes que serão criados
   - Após aprovação: execução sequencial das tarefas via Brainchain

3. **UI Visual — Design System Ereemby**
   - Cards grandes com progresso visual
   - Editor visual para acompanhar geração em tempo real
   - Seguir design system Liquid Glass Dark (Ereemby v2.0)

4. **Elementos de Progresso Estilo Lovable**
   - Barra de progresso com porcentagem
   - Lista de tasks com status: done ✅, in_progress ⏳, pending ⏸
   - Animação de relógio/spinner para tarefas em andamento
   - Contagem de ferramentas usadas (ex: "14 tools used")
   - Seções "DONE", "MORE WORK NEEDED", "NEXT"
   - Cards expansíveis com detalhes de cada tarefa
   - Botões "Details" e "Preview"
   - Indicador "Thinking" com tempo (ex: "Thought for 14s")
   - Tabela de "Files fixed" com File/Issue/Fix

### Referência Visual
- Screenshots do Lovable editor (progresso, tasks, status)
- Arquivo HTML de referência: `cirius-editor.html` (design tokens, layout split-pane, terminal, sidebar)

### Stack
- Frontend: React + Tailwind (design system Ereemby)
- Backend: Edge Functions (Brainchain + OpenRouter fallback)
- Realtime: Supabase channels para progresso live
