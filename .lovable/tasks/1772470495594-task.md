## Tarefa: Correção completa do pipeline Cirius

### Descrição
Implementada a correção completa do pipeline de geração do Cirius conforme o fluxo descrito:

1. **Prompt → PRD** (já funcionava): `build_prompt` gera PRD via AI Gateway/OpenRouter
2. **PRD → Orchestrator** (já funcionava): `generate_code` cria projeto no orquestrador com `lovable_project_id` do pool Brainchain
3. **Tasks sequenciais** (já funcionava): `orchestrator-tick` despacha tasks via `execute_next` → `executeTaskViaBrainchain`
4. **Polling de completion** (já funcionava): `orchestrator-tick` verifica `latest-message` para detectar conclusão
5. **Markdown mining** (já funcionava): `syncLatestMarkdownFiles` extrai código do markdown e merge em `source_files_json`
6. **★ NOVO: Refinamento por IA** — Após todas as tasks completarem, `autoCapture` agora dispara `triggerRefinement` que chama `cirius-generate?action=refine`. A função `refineSourceFiles` envia todos os arquivos para AI Gateway/OpenRouter para corrigir imports, exports, tipos e dependências.
7. **Deploy** (já funcionava): `cirius-deploy` faz push para GitHub/Vercel/Netlify/Supabase

### Alterações
- `cirius-generate/index.ts`: Adicionada função `refineSourceFiles()` e action `refine`
