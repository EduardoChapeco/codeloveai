## Tarefa: Brain não deve enviar sem projeto ativo

### Descrição
O Star AI tentava enviar mensagens mesmo sem um Brain ativo no workspace, auto-criando projetos silenciosamente. Isso causava erros quando o workspace não correspondia à conta integrada.

### Correções Aplicadas
1. **Removido auto-create no `send`**: A ação `send` agora retorna erro `brain_inactive` se nenhum brain ativo existir, em vez de criar um novo silenciosamente.
2. **Account mismatch retorna erro claro**: Se o brain existe mas não é acessível na conta Lovable atual, deleta o registro e retorna `brain_inactive` com mensagem pedindo reativação.
3. **Frontend trata `brain_inactive`**: O código do chat detecta o código `brain_inactive` e volta à tela de ativação do Star AI.

### Arquivos Alterados
- `supabase/functions/loveai-brain/index.ts` — send não auto-cria mais, retorna erro claro
- `src/pages/Brain.tsx` — trata código `brain_inactive`
