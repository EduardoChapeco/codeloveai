### Descrição
O fluxo de criação do Brain foi refatorado para usar o padrão Ghost Create corretamente:

### Correções Aplicadas
1. **Ghost Create com payload mínimo**: A criação do projeto agora usa `{ name, initial_message: "setup", visibility: "private" }` — payload mínimo sem objetos complexos
2. **Cancel imediato**: O `message_id` retornado pela API é capturado e cancelado imediatamente via `POST /projects/{id}/chat/{msgId}/cancel` para custo zero de créditos
3. **Injeção de config pós-criação**: Arquivos `brain-config.md` e `brain-output.json` são injetados via `edit-code` após o cancel, sem custo
4. **Prompts reforçados**: Todos os prompts agora incluem instrução explícita de não criar componentes/páginas e escrever apenas no `brain-output.json`
5. **Captura de resposta com fallback MD**: `captureResponse` agora verifica também `.lovable/tasks/brain-response.md` como alternativa ao JSON
6. **Lock anti-stale melhorado**: Locks de criação expiram após 2 minutos com leitura direta do `created_at`
