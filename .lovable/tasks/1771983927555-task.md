## Tarefa: Fix send-message payload

### Descrição
A Edge Function `send-message` v2.0 estava com payload incompleto — faltavam os campos `files`, `optimisticImageUrls`, `selected_elements` e `debug_mode` que a API do Lovable exige para reconhecer o intent `security_fix_v2` como gratuito. Sem esses campos, a API tratava todas as mensagens como modo normal (com consumo de créditos/plano).

### Correção
Adicionados os campos obrigatórios ao payload enviado para `api.lovable.dev/projects/{id}/chat`, restaurando o comportamento gratuito do `security_fix_v2`.
